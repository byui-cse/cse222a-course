Task 1

Situation

You are working for a company that needs a technology solution to determine how many of what types of programmers they need to hire. They have data in a database that contains all pertinent information regarding all employees for the company. The company also has a series of rules that apply to programmers and their development leads. They are;

  1. No programmer can have more than two teammates.
  2. The type the managers being tracked in the data set is a Development Lead.
  3. Development leads can manage no more than 15 programmers and no less than 3.
  4. All programmer teammates must be managed by the same development lead.
  5. Each programmer must have at least one teammate that has worked for the company for more than 1 year and has at least 5 years experience.
  6. No programmer can be a member of more than one team.

Using classes, design and create a code set that indicates what types of programmers are needed to meet the criteria above. Use classes and apply only IS_A inheritance. You will not have access to the employee database, so to test your application the Quality Assurance group has asked that you generate a testing data set to run your code against. this data set is to consist of 52 development leads and between 120 and 125 programmers. You are instructed to create the data set in such a way that the experience constraint is violated.

As part of the code set required by your employers,

  • create a function that assigns programmers to development leads based on the criteria stated, and
  • create a function using the result of the assignment function. Name this function generateNeededHires. It must return data describing the types of programmers that need to be hired.

Task 2

Recreate your solution for Task 1 using classes but this time use only HAS_A type inheritance.

Task 3

Recreate your solution for Task 1 using structures. This time choose between using either or both IS_A and HAS_A inheritance. Use your best judgment to make these decisions.

Review

After completing the previous steps:

  1. Review one of the many possible solutions for this task,
  2. Compare and contrast your code with the possible solution's code, and
  3. Make note of any improvements you could make for future tasks.
