Task 1

Setup Your Environment and Explore It

You will be setting up your machines to write command-line applications. This is a language course, not a platform course. There is a mobile development course that you can take after this one where you get the opportunity to apply what you learn in this course when writing mobile apps. We look forward to seeing you in that course with the understanding of Swift you get in this course. The tasks in this course are designed to prepare you for the mobile course.

Windows Users

Windows users should follow the Windows Install Instructions

Creating a Swift Project

Create a directory on your computer that will eventually hold individual directories for each of the tasks. If a week has more than one task you will need a separate directory that represents a Swift project for each task. Therefore create directories in you course directory for week01_task_1 and week01_task_2. In the week01_task_1 directory, follow the instructions for 'Building an Executable' section in this tutorial. You will need to use your terminal app to do this.

Run VSCode and select File->Open Workspace and navigate to the task directory. Open the file called main.swift in the Sources directory. Add the following code to the file then save the file. 

import Foundation

print("Installation test passed!")

Run the new code set as described in the 'Building an Executable' section of the tutorial.

macOS Users

The industry standard for developing Swift applications on macOS is Xcode. You can install Xcode through this link.

Creating a Swift Project

For this class we are creating apps to help you learn the language before you start to learn the libraries specific to the iOS platform. That's why you will be creating command line apps using Swift. This also helps you understand that Swift is a platform independent language, not something specific to Apple products. So let's create an app.

Following these steps will guarantee your setup is ready to go.
  1. Press command-Shift-N
  2. Select "macOS" in the window's top menu bar.
  3. Select "Command Line Tool."
  4. Enter week01_task_1 into the product name field.
  5. Select "Swift" from the "Language" pull down menu at the bottom of the window.
  6. Save the project anywhere you want. If I were you, I'd create a directory to hold each of the projects for the tasks for each of the weeks.
  7. In the left hand pane, select the "main.swift" file.
  8. Change "Hello, World!" to "Installation test passed!"
  9. Select command-r to compile and run the application. You will see "Installation test passed!" in the console on the lower right of the window.

Linux Users

Linux install instructions are available at this link. Most common distributions are supported. You will first need to download and install the dependencies. After you have downloaded the dependencies, you can download the current version of Swift at this link.

After you have successfully downloaded the Swift toolchain, you will need to add it to your PATH. The following is a general command that will work, but could vary depending on your download location and the shell you are currently using. 

echo "export PATH=/path/to/swift/directory/usr/bin:$PATH" >> ~/.bashrc
source ~/.bashrc

Task 2

Explore Some Code

A Swift file with some code that does something similar to code you've written in the past has been provided. Follow the instructions in the 'Creating a Swift Project' section above for your platform to create a Swift project called week01_task_2. Paste the code from this file into the main.swift file in the your project (in the Sources directory for Windows and Linux users). Then compile and run it. The code you pasted into main.swift will help you begin to explore the Swift language's syntax. Don't be too worried if you don't completely understand all the code right now. The things you see there will be covered in more detail later in the class. It would be good for you if you expressed your curiosity and started using web searches to figure out some of the syntax you see.

Stretch Challenge

Debugging your code for this and later classes will be much easier if you know how to use the stepwise debugger. This challenge will make your creation of code for this class much easier. If you learn how to use it now, it will save you hours of debugging later.

For Windows and Linux Users

Use the instructions in the tutorial for using the debugger as a template to stepwise debug the source code in Task 2. Set a breakpoint on line 13 of the source code and explore how the variables change and how functions are called. Make sure you 'step into' the functions, not just see what they return.
